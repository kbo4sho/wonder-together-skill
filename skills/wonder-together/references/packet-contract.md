# Wonder Together Packet Contract

Use this contract for every parent-facing packet.

## Required Inputs

- Child's question or topic
- Child's age
- Optional child name
- Optional parent context

If age or question is missing, ask for it before producing a full packet.

## Required Output

```markdown
# Wonder Together Packet

## Core idea
One or two child-sized sentences.

## Story to read together
A short grounded read-aloud scene with a child and grownup.

## Explain it simply
The answer in age-appropriate language.

## Visuals
The useful visual set, the best first visual, and optional follow-up visual commands or prompts.

## Try it together
One safe parent-led activity, drawing, pretend play, conversation script, or mini game.

## Parent note
What kids may misunderstand and how the grownup can respond.

## Follow-up questions
Three curiosity prompts.
```

## Format Selection

- Always include story and explanation.
- Add a diagram when a picture, motion, scale, sequence, body map, system map, or before/after comparison makes the idea click.
- Add an activity or experiment only when the idea is physical, safe, and room-scale.
- Add pretend play when roles, relationships, body systems, or flows can be acted out.
- Add a mini game when the child can safely sort examples, protect something, collect clues, or make fast choices.
- Add a misconception note when kids commonly form a sticky wrong model or the parent needs careful wording.
- Add history only when the human discovery story adds awe or clarity.

## Tone

- Warm, concrete, conversational.
- Parent-facing, not child-chatbot-facing.
- Useful enough for a grownup to read or use immediately.
- Honest without overexplaining.
- Playful only when the topic can hold play.

## Avoid

- Generic lectures.
- Generic tiny mascots, named particles, or magic explanations that replace the real concept.
- Every-format packets when fewer sections would be kinder.
- Claims that Wonder Together stores accounts, creates private links, charges money, or requires a backend.
