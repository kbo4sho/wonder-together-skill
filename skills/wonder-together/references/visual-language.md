# Wonder Together Visual Language

Wonder Together visuals should help a parent and child understand one section of a packet. They are not brand posters.

## Original Style

- Warm parent-child scenes, everyday rooms, windows, sidewalks, kitchen tables, libraries, parks, and car rides.
- Optional consistent `Big Wonderer` and `Little Wonderer` diagram actors when the output needs recurring characters. See [character-system.md](character-system.md).
- Simple diagrams that a child can inspect: arrows, paths, layers, comparisons, maps, timelines, and before/after panels.
- Soft field-guide colors: ink, cream, sky blue, leaf green, warm yellow, clay, berry, and muted red for caution.
- Hand-drawn linework, slightly imperfect but clear.
- Spacious composition with one main idea per image.
- Labels can be suggested, but the text explanation should live outside the image.

## Visual Types

- Story scene: a cozy moment from the read-aloud story.
- Explanation diagram: the clearest spatial, causal, scale, or sequence visual.
- Activity/game visual: setup, board, prop, drawing prompt, or conversation card.
- Map/timeline/object reference: one inspectable reference for history, geography, nature, body, or tools.
- Character sheet: only when defining or testing visual consistency, not as a default packet output.

## First Visual Rule

For ordinary wonder questions, choose exactly one first visual:

1. Explanation diagram, when the concept needs a picture.
2. Activity/game visual, when doing the activity matters more.
3. Story scene, when the packet is mostly emotional or narrative.
4. Optional overview only if the user explicitly asks.

Then list short follow-up commands such as:

- `Generate the explanation diagram.`
- `Generate the activity visual.`
- `Generate the story scene.`

## Do Not

- Copy, adapt, or imitate the Xiaohei character or blob-figure identity.
- Create a recurring Wonder Together mascot that takes over the explanation; if using Big/Little Wonderer, keep them as quiet diagram actors.
- Generate generic posters, covers, title cards, product images, worksheets, or overview graphics unless asked.
- Put the full packet inside an image.
- Depend on exact words inside generated images.
- Force visuals for parent notes or follow-up questions.
