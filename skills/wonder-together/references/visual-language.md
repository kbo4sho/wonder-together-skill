# Wonder Together Visual Language

Wonder Together visuals should help a parent and child understand one idea fast.
The default output is one visual-first, single-scene explainer, not a brand
poster and not a full packet.

## Original Style

- Warm parent-child scenes, everyday rooms, windows, sidewalks, kitchen tables, libraries, parks, and car rides.
- Consistent `Big Wonderer` and `Little Wonderer` diagram actors as the default parent-child anchor for ordinary explainers. See [character-system.md](character-system.md).
- Simple diagram cues that a child can inspect inside one scene: arrows, paths,
  sightlines, layers, scale cues, maps, timelines, and comparisons.
- Soft field-guide colors: ink, cream, sky blue, leaf green, warm yellow, clay, berry, and muted red for caution.
- Hand-drawn linework, slightly imperfect but clear.
- Spacious composition with one main idea per image.
- A few short in-image labels can name the mechanism when useful, but the
  generated image should not depend on long or exact text.

## Visual Types

- Single-scene explanation: the clearest spatial, causal, scale, or sequence
  visual folded into one coherent moment.
- Activity/game visual: setup, board, prop, drawing prompt, or conversation card.
- Map/timeline/object reference: one inspectable reference for history, geography, nature, body, or tools.
- Relationship scene: parent-child shared gaze or guidance when the question is emotional or wonder-based.
- Character sheet: only when defining or testing visual consistency, not as a default packet output.

## One Image Rule

For ordinary wonder questions, choose exactly one image:

1. Single-scene explanation, when the concept needs a picture.
2. Activity/game visual, when doing the activity matters more.
3. Relationship scene, when the concept is mostly emotional or narrative.
4. Optional overview only if the user explicitly asks.

If image generation is unavailable, provide exactly one polished prompt instead.
Mention optional follow-up images only in `Want more?`.

## Do Not

- Copy, adapt, or imitate the Xiaohei character or blob-figure identity.
- Create a recurring Wonder Together mascot that takes over the explanation; Big/Little Wonderer should reinforce the parent-child relationship while the concept remains central.
- Generate generic posters, covers, title cards, product images, worksheets, or overview graphics unless asked.
- Put the full packet inside an image.
- Split the default image into numbered panels, storyboards, before/after
  boards, or multiple separate ideas.
- Depend on long or exact words inside generated images.
- Generate multiple images by default.
